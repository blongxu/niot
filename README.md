## 👋 项目介绍

N-Iot 是“白龙须”旗下的一款物联网设备管理系统，包含工业物联网平台与智能边缘网关两大应用，专为工业环境设计的物联网数据采集与处理解决方案。该系统通过软硬件结合的方

式，有效解决了工业现场中设备协议多样化、接口不统一的数据集成挑战，实现了设备数据的统一接入、边缘处理和云端管理。

## ✨ 核心功能

### 📜 多协议设备接入与协议转换

      **支持十几种主流工业协议解析，通过智能边缘网关实现串口设备与以太网的协议转换，兼容各种智能工业场景，目前支持协议包含以下**

-   **标准工业协议**：Modbus RTU、Modbus TCP、BACnet/IP、OPC DA、OPC UA、IEC104、MQTT 等。
-   **PLC专用协议**：西门子PLC的S7协议、三菱PLC的MC协议等
-   **特殊用途驱动**：模拟器
-   **定制开发驱动**：消防主机

### 🖥️ 分布式站点与设备管理

      **提供站点创建、网关配置、设备分组的全流程管理，支持批量设备导入与继承配置，可通过模板快速部署多站点监控网络。**

-   **批量导入导出**：支持通过Excel模板批量导入设备信息，也可导出全部站点设备详细信息，便于线下处理与备份
-   **设备分组策略**：可按设备类型、功能区域、业务系统等维度对设备进行逻辑分组，支持跨站点设备统一分组管理
-   **地理位置标识**：可为每个站点设置经纬度信息，结合GIS地图实现站点可视化定位，便于快速查看分布式站点地理分布

### 🌐 实时数据采集与边缘计算

       **边缘网关本地实时采集设备点位数据，支持毫秒级轮询配置，结合本地计算规则实现异常值过滤与预处理，降低云端传输压力。**

-   **高频轮询机制**：支持毫秒级数据轮询配置，针对关键业务数据可设置更高的采集频率，确保数据实时性
-   **采集状态监控**：实时监测各采集点的工作状态与通信质量，采集异常时自动触发诊断流程，定位故障原因
-   **本地计算规则**：在边缘网关部署轻量级计算规则，实现数据滤波、异常检测、统计分析等预处理操作
-   **低延时实时处理**：边缘计算将处理任务放在网络边缘侧，减少数据往返云端的延迟，满足实时性要求高的业务场景

### 🚨 自定义事件触发与告警机制

       **基于固定值或系统变量创建事件触发器，支持多级告警分级（信息/告警/故障）与消息模板自定义，确保异常状态及时响应。**

-   **多类型触发条件**：支持基于固定阈值、动态基线、系统变量等多种条件定义事件触发器
-   **告警级别定义**：将告警分为信息、警告、故障三个等级，不同级别对应不同的处理流程和响应时限
-   **可定制消息模板**：提供消息模板自定义功能，根据不同告警类型和级别定制通知内容格式

### 🔁 开放API与系统集成能力

       **提供标准化设备属性读写API接口，支持HTTP/MQTT协议数据交互，可与第三方系统（如MES、SCADA）无缝集成，实现数据双向流动。**

-   **RESTful API**：提供符合REST规范的API接口，支持JSON格式数据交互，便于各种上层应用调用
-   **设备属性读写**：提供完整的设备属性读写接口，支持实时数据查询、历史数据检索、命令下发等功能
-   **访问权限控制**：基于角色的访问控制机制，精细控制不同系统对数据的读写权限

### 📊 功能对比 

-   **核心功能比较**

|     |     |     |
| --- | --- | --- |
| **对比维度** | **N-Iot** | **传统物联网产品** |
| **边缘计算能力** | 网关内置协议解析与边缘规则引擎，支持本地数据过滤、阈值判断与异常上报 | 依赖云端计算，需全量上传原始数据，网络带宽占用高，延迟≥100ms |
| **设备接入效率** | 可视化驱动配置界面，支持主流协议一键导入，批量设备模板继承配置 | 需手动编写协议解析脚本，单设备配置平均耗时30分钟，多设备场景需重复操作 |
| **系统安全性** | 站点-网关-设备三级密钥认证，支持设备证书双向校验，符合等保三级要求 | 多采用单一API令牌认证，设备接入权限颗粒度粗，存在越权访问风险 |
| **拓扑管理灵活性** | 支持物理位置/功能类型/业务流程多维分组，可视化设备关联与批量操作 | 多采用扁平列表管理，设备分组维度单一，跨分组操作需重复筛选 |
